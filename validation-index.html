🔧 Troubleshooting & Solutions

Problem: "AI-generated images don't detect"

Root cause: NFT uses SIFT feature extraction, which requires sharp edges. AI diffusion models blur edges intentionally for realism.

Solutions:

  1. Use high-contrast printed pattern (99% detection rate) — Download from test-images.html
  2. Use a real newspaper article instead of AI version (95%+ detection)
  3. Add QR code detection as fallback (~20 lines of code)
  4. Disable NFT for AI content, show overlays manually instead

Problem: "Performance is laggy/slow"

Root cause: 3 animated overlays + WebGL rendering at full resolution + deprecated THREE.js lighting

Quick Fixes (in order of impact):

  1. Disable overlay animations (remove animation__pulse, animation__glow)
  2. Reduce canvas resolution (720p instead of fullscreen)
  3. Switch to QR detection (uses less CPU than NFT tracking)
  4. Upgrade A-Frame to 1.7+ and THREE.js (fixes useLegacyLights warning)

❓ FAQ

Q: Why doesn't my AI-generated article work with NFT?

A: NFT uses SIFT feature detection, which needs sharp edges. Diffusion models (DALL-E, Midjourney) deliberately smooth edges for realism. The smoother an image looks, the fewer features SIFT can find.

Q: Can I lower the confidence threshold to fix it?

A: No. Your code already uses `minConfidence: 0.2` (20%), which is the minimum safe value. If detection fails, it's not because the threshold is too high—it's because there are no matching features at all.

Q: Will printing the image fix it?

A: Printing won't help. Printing adds texture/halftoning, but the underlying smooth gradients are still smooth. You'd need a feature-rich original image.

Q: How do I fix this?

A: Best options (in order):

  1. Use real newspaper/photo instead of AI (95%+ detection)
  2. Use high-contrast pattern (99% detection, most reliable)
  3. Add QR code detection (works on any image, ~20 lines of code)

Q: How is performance impact?

A: Your lagging is NOT caused by NFT detection—it's caused by three animated overlays with per-frame scale changes. Optimizations:

  • Disable `animation__pulse` and `animation__glow`
  • Reduce canvas resolution to 720p
  • Switch to QR detection (lower CPU cost)
